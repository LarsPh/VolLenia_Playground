## Important interpretation

`cells` means the initial 3D scalar field `A(x,y,z) in [0,1]`, not a multi-channel cell type. It should reset the simulation state.

The imported cells should be center-padded into the target simulation volume. Do not stretch them to fill the target volume in the first version.

The project uses x-fastest layout:

```cpp
index = (z * ny + y) * nx + x;
```

## Converter script requirements

Create:

```text
scripts/convert_lenia3d_animals.py
```

Requirements:

- Pure Python, no Node.js dependency.
- Read `animals3D.js` and extract entries with both `params` and `cells`.
- Decode RLE using logic equivalent to Lenia3D `src/utils/load.js`.
- Parse fraction strings in `b`, for example `"1,3/4,7/12,11/12"`.
- Write raw little-endian float32 cell files.
- Write `configs/lenia3d_reference/animals.json`.
- Use slugified names for files.
- Support `--limit N` for quick testing.

Suggested output paths:

```text

assets/cells/lenia3d_reference/<slug>.f32
```

## Runtime model

Add:

```cpp
struct LeniaAnimalPreset {
    int id = -1;
    std::string slug;
    std::string code;
    std::string name;
    std::string cname;
    VolumeDesc cells_desc;
    std::filesystem::path cells_file;
    LeniaParams params;
};
```

Add a catalog loader:

```cpp
class LeniaAnimalCatalog {
public:
    void load(const std::filesystem::path& manifest_path);
    int count() const;
    const LeniaAnimalPreset& animal(int index) const;
};
```

Add a cell volume loader for `.f32` files. Loading input files on CPU is OK; do not read back full GPU volumes.

## Lenia parameter model

Extend `LeniaParams` to represent Lenia3D `kn` and `gn`:

```cpp
enum class KernelCoreType {
    PolynomialBump = 1,
    ExponentialBump = 2,
    Step = 3,
    Staircase = 4,
};

enum class GrowthFunctionType {
    Polynomial = 1,
    Gaussian = 2,
    Step = 3,
};
```

Use Lenia3D-compatible parameter aliases:

```text
R -> radius
T -> T
m -> mu
s -> sigma
b -> shell_weights
kn -> kernel core
gn -> growth function
```

At minimum implement:

```text
KernelCoreType::PolynomialBump: (4r(1-r))^4
GrowthFunctionType::Polynomial: 2 * max(0, 1 - (u-m)^2/(9s^2))^4 - 1
GrowthFunctionType::Gaussian:   2 * exp(-(u-m)^2/(2s^2)) - 1
```

Keep current Gaussian growth as an option.

## Applying imported cells

Add a method like:

```cpp
void LeniaSimulation::resetImportedCells(DeviceVolumeView source_cells);
```

Implementation:

- Clear the target state volume.
- Center source cells in target volume.
- Copy values with a CUDA kernel.
- Clamp to `[0, 1]`.
- Reset generation to 0.

Do not resample/stretch the source cells in the first version.
